## 什么是Github？

先说在前面：GitHub 不是一个看起来很高级、只有程序员才能用的网站。

你可以先把它理解成一个“项目仓库网站”。

很多人会把自己做的软件、工具、代码、脚本、插件、文档放在 GitHub 上。  
如果这个项目是公开的，那你就可以进去看，也可以把里面的东西下载到自己电脑上。

你不需要一上来就学什么代码、分支、提交、命令行。  
如果你只是想从 GitHub 下载东西，那其实只要会看页面、会找按钮就够了。

‍

![](/image/pasted-1781082932470-ec7215.png)

举个例子：你现在在的这个网站blog是SH\_ZMD私人的，但是这个网站的各种设计和交互本身其实是Github上一个开源（指代码公开可直接使用）项目，我下载好这些代码以后就在本地开了这个blog的后台，然后我把代码拉到我自己的Github仓库上以后通过Github部署到Vercel上，Vercel帮我生成域名和网址，并开放到网络，所以你们能看到我这个blog并不是我花钱买了域名和服务器，而是通过Github仓库搭建到了别人的服务器上

‍

## 如何使用Github？

首先你可以注册一个Github账号，直接用Geogle账号登陆注册即可，然后再完成Github那些不当人的考察视力的题即可注册好自己的Github账号（初步只从Github上下载东西不需要注册账号，所以这一步非必须）

‍

## 怎么看一个项目好不好？

答案是看他的star数量

‍

![](/image/pasted-1781083962297-34827d.png)

star就是点赞数，这个项目有7900的赞，算非常高了

不过必须要说的是star不是检验的唯一标准，因为可能项目刚出来大家都不知道或者这东西太小众了没人用所以star少

就像你喜欢的一些视频/up主也不都是百大对吧

还有star多也不一定就好用，比如好多不要笑挑战的up还是百大对吧 （只能证明用的人多，不一定是你就是好）
